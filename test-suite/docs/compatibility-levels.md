# Compatibility Levels

## Self-tested

Used when a project has executed a subset of checks or when one or more checks failed, preventing a stronger compatibility claim. This is the most common starting point for early self-hosted worker onboarding.

## Compatible

Used when the project passes the required schema, behavior, and audit categories defined by the reference suite.

## Certified

Used when the project passes the full reference conformance bundle, including reporting checks, and produces a reproducible compliance report.
