# Test Categories

## Schema tests
Validate that required Pawprint fields remain present and the public Pawprint shape stays minimal.

## Behavior tests
Exercise deny, approval-gated, simulated, and budget-exhausted behavior through the current Pawly flow.

## Audit tests
Confirm that decisions produce audit artifacts with required fields, event types, and outcomes.

## Reporting tests
Confirm that Pawly-generated reports and validation reports contain the fields required by this suite.

## Self-hosted onboarding tests
Confirm that a self-hosted Pawprint worker card can be paired with the mock invocation and healthcheck artifacts used by the local onboarding flow.
