# Role Boundaries

A good Pawprint example does not describe a generic “helpful AI assistant.” It describes a bounded organizational role.

## What role boundaries should show

Each example should make it obvious:

- who the worker is;
- what work is allowed;
- what work must pause for handoff;
- what work is prohibited; and
- how the worker should communicate.

## Why role boundaries matter

The examples in this directory are meant to teach that Pawprint is not primarily about model capability. It is about clear role boundaries at the execution edge. A realistic example should therefore make the “no” boundaries and handoff conditions concrete.
